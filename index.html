HTML5

Significa Hyper Text Markup Language, Es la combinacion de Lenguaje de Marcado e Hipertexto para construir sitios Web. El lenguaje de marcado se utiliza para definir el documento de texto dentro de etiquetas que definen la estructura de las paginas Web, el hipertexto define el enlace entre las Paginas Web,define el significado y la estructura del contenido Web. Los elementos de HTML5 son los componentes basicos de las paginas Web y estan representados por etiquetas llamadas TAGS Utiliza el navegador para manipular texto, imagenes y otros contenidos para mostrarlo en un formato requerido

Aplicaciones de HTML

  1. DeviantART muro: Una aplicación de dibujo basado en HTML 5 que funciona en todos los navegadores modernos, y se puede bucear y empezar a dibujar sobre un lienzo en blanco
  2. HootSuite: Panel de control social basado en HTML5 con algunas funciones muy útiles: geo-localización de Tweet, búsqueda, elección entre retweets con estilo nuevo y viejo, con Google Analytics, interfaz personalizado y mucho más.
  3. X-Worlds: Algo interesante para los niños, done puede presentar un dibujo de tu personaje favorito y ver que se transforman en una ilustración impresionante.
  4. Coolendar: Un calendario basado en HTML5, con interfaz simple y limpio y funciones útiles para la gestión eficaz del tiempo.
  5. Scribd: Una aplicación web que proporciona acceso en el navegador a todo tipo de documentos y libros electrónicos subidos por los usuarios.

Versiones de HTML

  1. HTML 2.0: En 1995 se publica el estándar HTML 2.0. A pesar de su nombre, HTML 2.0 es el primer estándar oficial de HTML, es decir, el HTML 1.0 no existió como estándar. HTML 2.0 no soportaba tablas.
  2. HTML 3.2: Se publicó en 1997 y es la primera recomendación de HTML publicada por el W3C (Consorcio internacional). Esta revisión incorporó los últimos avances de las páginas web desarrolladas hasta 1996, como applets de Java y texto que fluye alrededor de las imágenes.
  3. HTML 4.01: La última especificación oficial de HTML se publicó en diciembre de 1999 y se denomina HTML 4.01. Desde la publicación de HTML 4.01, el W3C se centró en el desarrollo del estándar XHTML.
  4. HTML 4.01 Strict:En este tipo de documentos podemos usar etiquetas HTML 4.01, pero no se aceptan etiquetas obsoletas, es decir, etiquetas propias de versiones más antiguas
  5. HTML 4.01 Transitional: En este tipo de documentos se pueden usar todas las etiquetas de todas las versiones de HTML. Usar esta variante de HTML plantea el interrogante de si es correcto permitir el uso de etiquetas obsoletas que podrían dejar de funcionar en las proximas versiones de los navegadores. Sin embargo, este es el estándar más usado, porque combina la posibilidad de usar etiquetas más antiguas y etiquetas más modernas
  6. HTML 4.01 Frameset: Este tipo de documentos tiene soporte para frames. Los frames son unos marcos a modo de pequeñas subventanas dentro de una misma página web que se usaban mucho hace unos años pero que hoy en día se usan cada vez menos.
